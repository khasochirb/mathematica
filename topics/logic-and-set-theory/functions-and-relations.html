Functions & Relations

Exploring the connections between sets through relations and functions

Introduction to Relations

In mathematics, a relation between two sets is a collection of ordered pairs containing one object from each set. If the ordered pair (x, y) belongs to a relation R, we write xRy or (x, y) ∈ R. Relations generalize the concept of connections or associations between objects and provide a formal foundation for analyzing various mathematical and real-world relationships.

Binary Relations

A binary relation R from set A to set B is a subset of the Cartesian product A × B. In other words, it is a collection of ordered pairs (a, b) where a ∈ A and b ∈ B.

Special Types of Relations:

  • Reflexive Relation: For all a ∈ A, (a, a) ∈ R. Every element is related to itself.
  • Symmetric Relation: For all a, b ∈ A, if (a, b) ∈ R, then (b, a) ∈ R. If a is related to b, then b is related to a.
  • Antisymmetric Relation: For all a, b ∈ A, if (a, b) ∈ R and (b, a) ∈ R, then a = b. If a and b are mutually related, they must be the same element.
  • Transitive Relation: For all a, b, c ∈ A, if (a, b) ∈ R and (b, c) ∈ R, then (a, c) ∈ R. If a is related to b and b is related to c, then a is related to c.

Equivalence Relations

An equivalence relation is a binary relation that is reflexive, symmetric, and transitive. Equivalence relations define ways to partition a set into distinct, non-overlapping subsets called equivalence classes.

Example: Modular Arithmetic as an Equivalence Relation

Consider the relation R on the set of integers Z defined by:

a R b if and only if a ≡ b (mod 3) (or equivalently, if a - b is divisible by 3)

This relation is:

Reflexive: For any integer a, a - a = 0 is divisible by 3, so a R a.

Symmetric: If a R b, then a - b is divisible by 3. This means b - a = -(a - b) is also divisible by 3, so b R a.

Transitive: If a R b and b R c, then a - b and b - c are both divisible by 3. Their sum (a - b) + (b - c) = a - c is also divisible by 3, so a R c.

Since R is reflexive, symmetric, and transitive, it's an equivalence relation. It partitions Z into three equivalence classes: {3k}, {3k + 1}, and {3k + 2} for all integers k.

Partial Orders

A partial order is a binary relation that is reflexive, antisymmetric, and transitive. Partial orders represent ranking or hierarchical relationships where not all elements are comparable.

Example: Subset Relation as a Partial Order

Consider the subset relation ⊆ on the power set P(S) of some set S. This relation is:

Reflexive: Every set is a subset of itself, so A ⊆ A for all A ∈ P(S).

Antisymmetric: If A ⊆ B and B ⊆ A, then A = B.

Transitive: If A ⊆ B and B ⊆ C, then A ⊆ C.

Since ⊆ is reflexive, antisymmetric, and transitive, it's a partial order on P(S).

Functions as Special Relations

A function is a special type of relation that assigns to each element of a set, exactly one element of another set (or the same set). More formally, a function f from a set A to a set B, denoted f: A → B, is a relation from A to B where every element in A is related to exactly one element in B.

Key Function Terminology

  • Domain: The set A of all input values for the function.
  • Codomain: The set B that contains all possible output values of the function.
  • Range: The subset of B that consists of all actual output values of the function. The range is always a subset of the codomain.
  • Image: For an element a ∈ A, the image f(a) is the element in B that a maps to.
  • Preimage: For an element b ∈ B, the preimage f⁻¹(b) is the set of all elements in A that map to b.
Example: Analyzing a Simple Function

Let f: R → R be defined by f(x) = x².

Domain: R (all real numbers)

Codomain: R (all real numbers)

Range: [0, ∞) (non-negative real numbers), which is a proper subset of the codomain

Image of 3: f(3) = 3² = 9

Preimage of 4: f⁻¹(4) = {-2, 2} (both -2 and 2 map to 4)

Properties of Functions

Injective (One-to-One) Functions

A function f: A → B is injective if distinct elements in A map to distinct elements in B. Formally, for all a₁, a₂ ∈ A, if a₁ ≠ a₂, then f(a₁) ≠ f(a₂). Equivalently, if f(a₁) = f(a₂), then a₁ = a₂.

Example: Testing Injectivity

Let's analyze if these functions are injective:

1. f(x) = 2x + 3 from R to R

If f(a) = f(b), then 2a + 3 = 2b + 3, which implies a = b. So f is injective.

2. g(x) = x² from R to R

g(-2) = 4 and g(2) = 4, but -2 ≠ 2. So g is not injective.

3. h(x) = x² from [0, ∞) to R

If h(a) = h(b) for a, b ≥ 0, then a² = b², which implies a = b. So h is injective.

Surjective (Onto) Functions

A function f: A → B is surjective if every element in B is the image of at least one element in A. Formally, for all b ∈ B, there exists a ∈ A such that f(a) = b.

Example: Testing Surjectivity

Let's analyze if these functions are surjective:

1. f(x) = 2x + 3 from R to R

For any y ∈ R, we can find x = (y - 3)/2 such that f(x) = y. So f is surjective.

2. g(x) = x² from R to R

No x ∈ R can map to -1 ∈ R, since x² ≥ 0 for all x. So g is not surjective.

3. h(x) = x² from R to [0, ∞)

For any y ∈ [0, ∞), we can find x = √y such that h(x) = y. So h is surjective.

Bijective Functions

A function is bijective if it is both injective and surjective. In a bijective function, each element of the codomain is paired with exactly one element of the domain. Bijective functions establish a one-to-one correspondence between the elements of two sets.

Example: Bijective Function

The function f(x) = 2x + 3 from R to R is bijective because:

- It's injective: If f(a) = f(b), then 2a + 3 = 2b + 3, which implies a = b.

- It's surjective: For any y ∈ R, x = (y - 3)/2 satisfies f(x) = y.

The function g(x) = x² from [0, ∞) to [0, ∞) is also bijective because:

- It's injective: If g(a) = g(b) for a, b ≥ 0, then a² = b², which implies a = b.

- It's surjective: For any y ∈ [0, ∞), x = √y satisfies g(x) = y.

Function Composition and Inverse Functions

Function Composition

Given functions f: A → B and g: B → C, the composition of g and f, denoted g ∘ f: A → C, is defined by (g ∘ f)(a) = g(f(a)) for all a ∈ A.

Example: Function Composition

Let f(x) = x² + 1 and g(x) = 3x - 2. Then:

(g ∘ f)(x) = g(f(x)) = g(x² + 1) = 3(x² + 1) - 2 = 3x² + 3 - 2 = 3x² + 1

(f ∘ g)(x) = f(g(x)) = f(3x - 2) = (3x - 2)² + 1 = 9x² - 12x + 4 + 1 = 9x² - 12x + 5

Note that g ∘ f ≠ f ∘ g, showing that function composition is generally not commutative.

Inverse Functions

A function f: A → B has an inverse if and only if it is bijective. The inverse function, denoted f⁻¹: B → A, is defined by f⁻¹(b) = a if and only if f(a) = b.

Properties of inverse functions:

  • f⁻¹(f(a)) = a for all a ∈ A
  • f(f⁻¹(b)) = b for all b ∈ B
  • f⁻¹ ∘ f = IdA (the identity function on A)
  • f ∘ f⁻¹ = IdB (the identity function on B)
Example: Finding Inverse Functions

Let's find the inverse of f(x) = 2x + 3.

Step 1: Let y = 2x + 3

Step 2: Solve for x. x = (y - 3)/2

Step 3: Swap x and y. f⁻¹(x) = (x - 3)/2

To verify: (f⁻¹ ∘ f)(x) = f⁻¹(2x + 3) = ((2x + 3) - 3)/2 = 2x/2 = x

And: (f ∘ f⁻¹)(x) = f((x - 3)/2) = 2((x - 3)/2) + 3 = (x - 3) + 3 = x

Applications of Relations and Functions

Applications in Mathematics

  • Group Theory: Functions establish homomorphisms between algebraic structures.
  • Topology: Continuous functions preserve the structure of topological spaces.
  • Analysis: Functions are used to model rates of change, areas, and volumes.
  • Graph Theory: Relations are represented as edges in graphs.

Applications in Computer Science

  • Database Theory: Relations are the foundation of relational databases.
  • Algorithm Analysis: Functions describe the growth rate of algorithms.
  • Programming: Functions map inputs to outputs in a deterministic way.
  • Automata Theory: Transition functions define the behavior of finite state machines.

Applications in Real-World Modeling

  • Physics: Functions model relationships between physical quantities.
  • Economics: Functions describe supply, demand, and utility.
  • Social Networks: Relations represent connections between individuals.
  • Genetics: Functions map genotypes to phenotypes.
Example: Economic Modeling with Functions

Consider a simple economic model where the demand for a product depends on its price. Let's define the demand function D(p) that gives the quantity demanded at price p.

D(p) = 1000 - 5p for p ∈ [0, 200]

This function has:

- Domain: [0, 200] (the set of possible prices)

- Range: [0, 1000] (the set of possible demand quantities)

- At p = $50, D(50) = 1000 - 5(50) = 750 units are demanded

- If we observe a demand of 600 units, we can find the corresponding price by solving 600 = 1000 - 5p, which gives p = $80
