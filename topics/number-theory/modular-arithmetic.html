Modular Arithmetic

Understanding arithmetic operations with remainders and their applications in cryptography and computer science

Home » Resources » Mathematics Roadmap » Number Theory » Modular Arithmetic

Introduction to Modular Arithmetic

Modular arithmetic is a system of arithmetic for integers where numbers "wrap around" after reaching a certain value, called the modulus. This concept is analogous to a clock, where after reaching 12, we wrap back to 1.

Congruence Relation

The fundamental concept in modular arithmetic is congruence. We say that two integers a and b are congruent modulo n, written as:

a ≡ b (mod n)

if a and b leave the same remainder when divided by n. Equivalently, a ≡ b (mod n) if n divides (a - b).

Example: Congruence Relation

23 ≡ 8 (mod 5) because 23 and 8 both leave remainder 3 when divided by 5.

Alternatively, 23 - 8 = 15, which is divisible by 5.

Similarly, -7 ≡ 1 (mod 4) because -7 and 1 both leave remainder 1 when divided by 4 (note that -7 = -8 + 1 = -2×4 + 1).

Modular Equivalence Classes

For a given modulus n, integers can be grouped into n different equivalence classes. Each class contains all integers that are congruent to each other modulo n. These classes are:

[0], [1], [2], ..., [n-1]

where [r] represents all integers that leave remainder r when divided by n.

Example: Equivalence Classes for Modulo 4

[0] = {..., -8, -4, 0, 4, 8, 12, ...}

[1] = {..., -7, -3, 1, 5, 9, 13, ...}

[2] = {..., -6, -2, 2, 6, 10, 14, ...}

[3] = {..., -5, -1, 3, 7, 11, 15, ...}

Basic Operations in Modular Arithmetic

Modular arithmetic defines operations on congruence classes. These operations are well-defined because they respect the congruence relation.

Addition and Subtraction

For integers a, b, and modulus n:

(a + b) mod n = ((a mod n) + (b mod n)) mod n
(a - b) mod n = ((a mod n) - (b mod n)) mod n
Example: Modular Addition and Subtraction

Addition: (17 + 25) mod 7 = (17 mod 7 + 25 mod 7) mod 7 = (3 + 4) mod 7 = 7 mod 7 = 0

Subtraction: (17 - 25) mod 7 = (17 mod 7 - 25 mod 7) mod 7 = (3 - 4) mod 7 = -1 mod 7 = 6

Note: -1 ≡ 6 (mod 7) because -1 + 7 = 6

Multiplication

For integers a, b, and modulus n:

(a × b) mod n = ((a mod n) × (b mod n)) mod n
Example: Modular Multiplication

(17 × 25) mod 7 = (17 mod 7 × 25 mod 7) mod 7 = (3 × 4) mod 7 = 12 mod 7 = 5

Exponentiation

Modular exponentiation is computing ab mod n efficiently. The naive approach of computing ab first and then taking the modulo doesn't work well for large exponents due to the size of ab.

Instead, we can use the property:

ab mod n = ((a mod n)b) mod n

And for further efficiency, we can use the square-and-multiply algorithm:

a2b mod n = (ab mod n)2 mod n
ab+1 mod n = (ab mod n × a mod n) mod n
Example: Modular Exponentiation

Calculate 35 mod 7 using square-and-multiply:

31 mod 7 = 3

32 mod 7 = 31 × 31 mod 7 = 3 × 3 mod 7 = 9 mod 7 = 2

34 mod 7 = 32 × 32 mod 7 = 2 × 2 mod 7 = 4

35 mod 7 = 34 × 31 mod 7 = 4 × 3 mod 7 = 12 mod 7 = 5

Modular Inverses and Division

Modular Inverses

The modular multiplicative inverse of an integer a with respect to modulus n is an integer a-1 such that:

a × a-1 ≡ 1 (mod n)

A modular inverse exists if and only if a and n are coprime (their greatest common divisor is 1).

Example: Finding Modular Inverse

Find the modular inverse of 3 modulo 7.

We need to find x such that 3x ≡ 1 (mod 7).

Testing values:

3 × 1 = 3 ≡ 3 (mod 7)

3 × 2 = 6 ≡ 6 (mod 7)

3 × 3 = 9 ≡ 2 (mod 7)

3 × 4 = 12 ≡ 5 (mod 7)

3 × 5 = 15 ≡ 1 (mod 7)

Therefore, 3-1 ≡ 5 (mod 7)

Extended Euclidean Algorithm

The Extended Euclidean Algorithm provides an efficient method to find the modular inverse. It finds integers s and t such that:

as + nt = gcd(a, n)

If gcd(a, n) = 1, then s is the modular inverse of a modulo n.

Example: Extended Euclidean Algorithm

Find the modular inverse of 3 modulo 7 using the Extended Euclidean Algorithm:

7 = 3 × 2 + 1

3 = 1 × 3 + 0

Working backwards:

1 = 7 - 3 × 2

1 = 7 - 3 × 2 = 7(1) + 3(-2)

So s = -2, and the modular inverse of 3 modulo 7 is -2 ≡ 5 (mod 7).

Modular Division

In modular arithmetic, division is defined as multiplication by the modular inverse. For integers a, b, and modulus n, with gcd(b, n) = 1:

a/b mod n = (a × b-1) mod n

where b-1 is the modular inverse of b modulo n.

Example: Modular Division

Calculate 15/3 mod 7:

First, find 3-1 mod 7 = 5 (from previous example)

Then, 15/3 mod 7 = (15 × 5) mod 7 = 75 mod 7 = 5

Linear Congruences and Chinese Remainder Theorem

Linear Congruences

A linear congruence is an equation of the form:

ax ≡ b (mod n)

This equation has a solution if and only if gcd(a, n) divides b. If gcd(a, n) = d and d divides b, then there are d solutions modulo n.

Example: Solving Linear Congruence

Solve 4x ≡ 5 (mod 9):

gcd(4, 9) = 1, so there is a unique solution modulo 9.

Finding the modular inverse of 4 modulo 9:

4 × 7 = 28 ≡ 1 (mod 9), so 4-1 ≡ 7 (mod 9)

Multiplying both sides by 7:

4x × 7 ≡ 5 × 7 (mod 9)

x ≡ 35 ≡ 8 (mod 9)

Chinese Remainder Theorem

The Chinese Remainder Theorem (CRT) provides a way to solve a system of linear congruences when the moduli are pairwise coprime. For the system:

x ≡ a1 (mod m1)
x ≡ a2 (mod m2)
...
x ≡ ak (mod mk)

where m1, m2, ..., mk are pairwise coprime, there is a unique solution modulo M = m1 × m2 × ... × mk.

Example: Chinese Remainder Theorem

Solve the system:

x ≡ 2 (mod 3)

x ≡ 3 (mod 5)

x ≡ 2 (mod 7)

Using CRT:

M = 3 × 5 × 7 = 105

M1 = M/3 = 35, M2 = M/5 = 21, M3 = M/7 = 15

Find modular inverses:

35-1 ≡ 2 (mod 3), 21-1 ≡ 1 (mod 5), 15-1 ≡ 1 (mod 7)

Solution:

x ≡ (2 × 35 × 2 + 3 × 21 × 1 + 2 × 15 × 1) mod 105

x ≡ (140 + 63 + 30) mod 105 ≡ 233 mod 105 ≡ 23 (mod 105)

Applications of Modular Arithmetic

Cryptography

Modular arithmetic is fundamental to many cryptographic systems:

  • RSA Encryption: Based on the difficulty of factoring large numbers and modular exponentiation.
  • Diffie-Hellman Key Exchange: Uses modular exponentiation to establish a shared secret over an insecure channel.
  • Elliptic Curve Cryptography: Utilizes modular arithmetic in the context of elliptic curves.

Computer Science

Modular arithmetic has various applications in computer science:

  • Hash Functions: Modular operations are used to map data to fixed-size values.
  • Random Number Generation: Linear congruential generators use modular arithmetic to produce pseudo-random numbers.
  • Checksum Algorithms: Use modular operations to detect errors in data transmission.

Calendar Calculations

Modular arithmetic is used to determine days of the week, calculate leap years, and implement calendar systems.

Example: Day of the Week Calculation

Zeller's Congruence is a formula that calculates the day of the week for a given date:

h = (q + ⌊13(m+1)/5⌋ + K + ⌊K/4⌋ + ⌊J/4⌋ - 2J) mod 7

where h is the day of the week (0 = Saturday, 1 = Sunday, etc.), q is the day of the month, m is the month (3 = March, 4 = April, etc., with January and February counted as months 13 and 14 of the previous year), K is the year of the century (year mod 100), and J is the zero-based century (⌊year/100⌋).

Music Theory

Modular arithmetic with modulus 12 is used to represent the twelve notes in an octave and analyze chord structures and relations.
