Greatest Common Divisor (GCD)

Definition and Basic Properties

The Greatest Common Divisor (GCD) of two or more integers is the largest positive integer that divides each of the integers without a remainder. For integers a and b, not both zero, their GCD is denoted as gcd(a, b) or (a, b).

Key properties of the GCD:

  • gcd(a, 0) = |a| for a ≠ 0
  • gcd(a, a) = |a|
  • gcd(a, b) = gcd(b, a)
  • gcd(a, b) = gcd(a, -b) = gcd(-a, b) = gcd(-a, -b)
  • gcd(a, b) = gcd(a, b - a) for any integers a and b
  • If a divides both b and c, then a divides gcd(b, c)
  • gcd(a, b) × lcm(a, b) = |a × b|, where lcm is the least common multiple

Finding the GCD by Prime Factorization

One method to find the GCD is to use prime factorization. If:

a = p₁α₁ × p₂α₂ × ... × pkαk
b = p₁β₁ × p₂β₂ × ... × pkβk

Then:

gcd(a, b) = p₁min(α₁,β₁) × p₂min(α₂,β₂) × ... × pkmin(αk,βk)
Example: GCD by Prime Factorization

Find gcd(48, 180):

48 = 24 × 31

180 = 22 × 32 × 51

gcd(48, 180) = 2min(4,2) × 3min(1,2) × 5min(0,1) = 22 × 31 × 50 = 4 × 3 × 1 = 12

Euclidean Algorithm

The Euclidean Algorithm is an efficient method for computing the GCD of two integers, based on the principle that if a and b are integers with a ≥ b > 0, then gcd(a, b) = gcd(b, a mod b).

Algorithm Description

To find gcd(a, b) using the Euclidean Algorithm:

  1. If b = 0, return a as the GCD.
  2. Otherwise, compute a mod b and set a ← b, b ← a mod b.
  3. Repeat steps 1 and 2 until b = 0.
Example: Euclidean Algorithm

Find gcd(48, 180) using the Euclidean Algorithm:

Step 1: a = 180, b = 48

Step 2: a mod b = 180 mod 48 = 36

Step 3: a = 48, b = 36

Step 4: a mod b = 48 mod 36 = 12

Step 5: a = 36, b = 12

Step 6: a mod b = 36 mod 12 = 0

Step 7: Since b = 0, gcd(48, 180) = a = 12

Proof of Correctness

To prove that gcd(a, b) = gcd(b, a mod b), we need to show that any common divisor of a and b is also a common divisor of b and a mod b, and vice versa:

  • Let d be a common divisor of a and b. Then a = dm and b = dn for some integers m and n.
  • We can write a = bq + r, where r = a mod b. Therefore, r = a - bq = dm - dnq = d(m - nq).
  • Since r is divisible by d, any common divisor of a and b is also a divisor of r = a mod b.
  • Similarly, if d is a common divisor of b and r, then d divides a = bq + r, making d a common divisor of a and b.

Complexity Analysis

The Euclidean Algorithm has a time complexity of O(log(min(a, b))), making it very efficient even for large integers. This is because the values decrease rapidly in each iteration.

Extended Euclidean Algorithm

The Extended Euclidean Algorithm is an extension of the Euclidean Algorithm that computes, in addition to the GCD of integers a and b, the coefficients s and t such that:

as + bt = gcd(a, b)

This equation is known as Bézout's identity, and the integers s and t are called Bézout coefficients.

Algorithm Description

To find gcd(a, b) and the Bézout coefficients s and t using the Extended Euclidean Algorithm:

  1. If b = 0, then gcd(a, b) = a, s = 1, t = 0, and return (a, s, t).
  2. Otherwise, compute a mod b and the quotient q = ⌊a/b⌋.
  3. Recursively compute gcd(b, a mod b) and the corresponding coefficients s' and t'.
  4. Set s = t' and t = s' - q × t'.
  5. Return (gcd, s, t).
Example: Extended Euclidean Algorithm

Find gcd(120, 23) and Bézout coefficients:

Step 1: a = 120, b = 23

Step 2: 120 = 23 × 5 + 5

Step 3: 23 = 5 × 4 + 3

Step 4: 5 = 3 × 1 + 2

Step 5: 3 = 2 × 1 + 1

Step 6: 2 = 1 × 2 + 0

Working backwards:

1 = 3 - 2 × 1 = 3 - 1 × 2

1 = 3 - (5 - 3 × 1) × 1 = 3 - 5 + 3 = 2 × 3 - 5

1 = 2(23 - 5 × 4) - 5 = 2 × 23 - 8 × 5 - 5 = 2 × 23 - 9 × 5

1 = 2 × 23 - 9(120 - 23 × 5) = 2 × 23 - 9 × 120 + 45 × 23 = 47 × 23 - 9 × 120

Therefore, gcd(120, 23) = 1, s = -9, and t = 47

Verification: 120 × (-9) + 23 × 47 = -1080 + 1081 = 1

Applications of the Extended Euclidean Algorithm

  • Finding Modular Multiplicative Inverse: If gcd(a, m) = 1, then a-1 ≡ s (mod m), where s is the Bézout coefficient.
  • Solving Linear Diophantine Equations: Finding integer solutions to ax + by = c.
  • Chinese Remainder Theorem: Constructing a solution to a system of linear congruences.
  • Continued Fractions: Computing the best rational approximations of real numbers.

Least Common Multiple (LCM)

Definition and Basic Properties

The Least Common Multiple (LCM) of two or more integers is the smallest positive integer that is divisible by each of the integers. For integers a and b, their LCM is denoted as lcm(a, b).

Key properties of the LCM:

  • lcm(a, 0) = 0 for any a
  • lcm(a, a) = |a|
  • lcm(a, b) = lcm(b, a)
  • lcm(a, b) = lcm(a, -b) = lcm(-a, b) = lcm(-a, -b)
  • If a divides c and b divides c, then lcm(a, b) divides c
  • gcd(a, b) × lcm(a, b) = |a × b|

Finding the LCM by Prime Factorization

Using prime factorization, if:

a = p₁α₁ × p₂α₂ × ... × pkαk
b = p₁β₁ × p₂β₂ × ... × pkβk

Then:

lcm(a, b) = p₁max(α₁,β₁) × p₂max(α₂,β₂) × ... × pkmax(αk,βk)
Example: LCM by Prime Factorization

Find lcm(24, 36):

24 = 23 × 31

36 = 22 × 32

lcm(24, 36) = 2max(3,2) × 3max(1,2) = 23 × 32 = 8 × 9 = 72

Computing LCM Using GCD

A more efficient method for computing the LCM, particularly when the GCD is already known, is to use the formula:

lcm(a, b) = |a × b| / gcd(a, b)
Example: LCM Using GCD

Find lcm(24, 36) using the GCD formula:

gcd(24, 36) = 12 (using Euclidean Algorithm or prime factorization)

lcm(24, 36) = |24 × 36| / 12 = 864 / 12 = 72

Applications in Number Theory and Computer Science

Solving Linear Diophantine Equations

A Linear Diophantine Equation is an equation of the form ax + by = c, where a, b, and c are integers, and the solutions x and y are also required to be integers.

Such an equation has a solution if and only if gcd(a, b) divides c. If (x₀, y₀) is a particular solution, then all solutions are given by:

x = x₀ + (b/d)t
y = y₀ - (a/d)t

for any integer t, where d = gcd(a, b).

Example: Linear Diophantine Equation

Solve 15x + 25y = 10:

First, check if gcd(15, 25) divides 10:

gcd(15, 25) = 5, and 5 divides 10, so solutions exist.

Using the Extended Euclidean Algorithm to find s and t such that 15s + 25t = 5:

15 = 25 × 0 + 15

25 = 15 × 1 + 10

15 = 10 × 1 + 5

10 = 5 × 2 + 0

Working backwards:

5 = 15 - 10 × 1 = 15 - (25 - 15) = 2 × 15 - 25

So, 15 × 2 + 25 × (-1) = 5

Multiplying by 2: 15 × 4 + 25 × (-2) = 10

Therefore, a particular solution is x₀ = 4, y₀ = -2

All solutions are: x = 4 + 5t, y = -2 - 3t for any integer t

Modular Arithmetic and Congruences

The GCD and the Extended Euclidean Algorithm are essential in modular arithmetic, particularly for finding modular multiplicative inverses and solving linear congruences.

Computer Science Applications

  • Cryptography: GCD and Extended Euclidean Algorithm are used in RSA and other public-key cryptosystems.
  • Rational Number Representation: GCD is used to reduce fractions to their simplest form.
  • Computer Graphics: GCD is used in algorithms for drawing lines and curves on discrete grids.
  • String Algorithms: GCD can be used to find patterns in strings.
