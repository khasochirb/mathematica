Graph Representation

Understanding adjacency matrices, adjacency lists, and other graph representations

Home » Learning Resources » Mathematics Roadmap » Graph Theory » Graph Representation

Introduction to Graphs

A graph is a mathematical structure used to model pairwise relationships between objects. It consists of a set of vertices (also called nodes) and a set of edges that connect pairs of vertices. Graphs are versatile structures that can represent many real-world scenarios, from social networks to computer networks, from road maps to molecular structures.

Basic Terminology

  • Vertex (Node): A fundamental unit of a graph, often represented as a point.
  • Edge: A connection between two vertices, often represented as a line.
  • Adjacent: Two vertices are adjacent if there is an edge connecting them.
  • Incident: An edge is incident to a vertex if the vertex is one of the endpoints of the edge.
  • Degree: The number of edges incident to a vertex.

Types of Graphs

  • Undirected Graph: Edges have no direction, representing a symmetric relationship between vertices.
  • Directed Graph (Digraph): Edges have a direction, representing an asymmetric relationship from one vertex to another.
  • Weighted Graph: Edges have weights or costs associated with them.
  • Simple Graph: A graph with no self-loops (edges connecting a vertex to itself) and no multiple edges between the same pair of vertices.
  • Connected Graph: A graph where there is a path between every pair of vertices.
  • Complete Graph: A graph where every vertex is connected to every other vertex.
  • Bipartite Graph: A graph whose vertices can be divided into two disjoint sets such that every edge connects a vertex in one set to a vertex in the other set.
Example: Social Network as a Graph

Consider a social network where people are represented as vertices and friendships as edges:

  • Vertices: Alice, Bob, Charlie, Diana, Ethan
  • Edges: (Alice, Bob), (Alice, Charlie), (Bob, Diana), (Charlie, Diana), (Diana, Ethan)

This is an undirected graph because friendship is typically a mutual relationship. The graph is connected because there's a path between any two people. The degree of Diana is 3, as she is friends with Bob, Charlie, and Ethan.

Adjacency Matrix

An adjacency matrix is a square matrix used to represent a finite graph. The elements of the matrix indicate whether pairs of vertices are adjacent (connected by an edge) or not.

Definition

For a graph with n vertices, the adjacency matrix A is an n × n matrix where:

  • A[i, j] = 1 if there is an edge from vertex i to vertex j
  • A[i, j] = 0 if there is no edge from vertex i to vertex j

For weighted graphs, the value of A[i, j] can be the weight of the edge from vertex i to vertex j, with 0 or infinity (∞) indicating no edge.

Properties

  • Space Complexity: O(V²), where V is the number of vertices.
  • For Undirected Graphs: The adjacency matrix is symmetric, i.e., A[i, j] = A[j, i].
  • For Directed Graphs: The adjacency matrix may not be symmetric.
  • Self-loops: Represented by non-zero values along the diagonal.
  • Degree: For an undirected graph, the sum of the i-th row (or column) gives the degree of vertex i.

For an undirected graph with vertices {1, 2, 3, 4} and edges {(1,2), (1,3), (2,3), (3,4)}, the adjacency matrix is:

A = [0 1 1 0]

[1 0 1 0]

[1 1 0 1]

[0 0 1 0]

Advantages and Disadvantages

Advantages:

  • Simple to implement and understand.
  • Checking if there is an edge between two vertices takes O(1) time.
  • Efficient for dense graphs.
  • Matrix operations can be used to derive graph properties (e.g., A^n[i, j] gives the number of walks of length n from vertex i to j).

Disadvantages:

  • Wastes space for sparse graphs (those with few edges compared to the maximum possible).
  • Adding or removing a vertex requires resizing the matrix, which is inefficient.
Example: Implementing Adjacency Matrix in Code

Here's how you might implement an adjacency matrix for an undirected graph in Python:


def create_adjacency_matrix(vertices, edges):
    n = len(vertices)
    # Initialize matrix with zeros
    matrix = [[0 for _ in range(n)] for _ in range(n)]
    
    # Add edges to the matrix
    for u, v in edges:
        # Find the indices of the vertices in the vertex list
        i = vertices.index(u)
        j = vertices.index(v)
        # For undirected graph, set both entries
        matrix[i][j] = 1
        matrix[j][i] = 1
        
    return matrix

# Example usage
vertices = ['A', 'B', 'C', 'D']
edges = [('A', 'B'), ('A', 'C'), ('B', 'C'), ('C', 'D')]
adj_matrix = create_adjacency_matrix(vertices, edges)
for row in adj_matrix:
    print(row)

Output:


[0, 1, 1, 0]
[1, 0, 1, 0]
[1, 1, 0, 1]
[0, 0, 1, 0]

Adjacency List

An adjacency list is a collection of unordered lists used to represent a finite graph. Each list describes the set of neighbors of a vertex in the graph.

Definition

For a graph with vertices {v₁, v₂, ..., vₙ}, an adjacency list consists of n lists, where the i-th list contains all the vertices adjacent to vᵢ.

Properties

  • Space Complexity: O(V + E), where V is the number of vertices and E is the number of edges.
  • For Undirected Graphs: If vertex j appears in the adjacency list of vertex i, then vertex i appears in the adjacency list of vertex j.
  • For Directed Graphs: The adjacency list of vertex i contains all vertices j such that there is an edge from i to j.
  • Weighted Graphs: Each entry in the adjacency list can include both the neighboring vertex and the weight of the edge.

For an undirected graph with vertices {1, 2, 3, 4} and edges {(1,2), (1,3), (2,3), (3,4)}, the adjacency list is:

1: [2, 3]

2: [1, 3]

3: [1, 2, 4]

4: [3]

Advantages and Disadvantages

Advantages:

  • Space-efficient for sparse graphs.
  • Adding a vertex is easy (just add a new list).
  • Iterating over all edges of a vertex is efficient.

Disadvantages:

  • Checking if there is an edge between two vertices takes O(degree) time in the worst case.
  • Less efficient for dense graphs compared to adjacency matrix.
Example: Implementing Adjacency List in Code

Here's how you might implement an adjacency list for an undirected graph in Python:


def create_adjacency_list(vertices, edges):
    # Initialize empty lists for each vertex
    adj_list = {vertex: [] for vertex in vertices}
    
    # Add edges to the adjacency list
    for u, v in edges:
        # For undirected graph, add both directions
        adj_list[u].append(v)
        adj_list[v].append(u)
        
    return adj_list

# Example usage
vertices = ['A', 'B', 'C', 'D']
edges = [('A', 'B'), ('A', 'C'), ('B', 'C'), ('C', 'D')]
adj_list = create_adjacency_list(vertices, edges)
for vertex, neighbors in adj_list.items():
    print(f"{vertex}: {neighbors}")

Output:


A: ['B', 'C']
B: ['A', 'C']
C: ['A', 'B', 'D']
D: ['C']

Other Representations

Edge List

An edge list is a simple representation that stores a graph as a list of all its edges. Each edge is typically represented as a pair (or triplet for weighted graphs) of vertices.

Properties:

  • Space Complexity: O(E), where E is the number of edges.
  • Simple to implement but inefficient for most graph operations.
  • Useful when the primary operation is to iterate over all edges.

For a graph with edges {(1,2), (1,3), (2,3), (3,4)}, the edge list is:

[(1,2), (1,3), (2,3), (3,4)]

Incidence Matrix

An incidence matrix is a rectangular matrix that shows the relationship between vertices and edges. For a graph with n vertices and m edges, the incidence matrix is an n × m matrix where:

  • For undirected graphs, M[i, j] = 1 if vertex i is incident to edge j, and 0 otherwise.
  • For directed graphs, M[i, j] = 1 if edge j is directed from vertex i, M[i, j] = -1 if edge j is directed to vertex i, and 0 otherwise.

Properties:

  • Space Complexity: O(V × E), where V is the number of vertices and E is the number of edges.
  • Useful for certain graph algorithms and when working with the dual graph.
  • Not commonly used for general graph algorithms due to its space inefficiency.
Example: Incidence Matrix

Consider an undirected graph with vertices {1, 2, 3, 4} and edges e₁ = (1,2), e₂ = (1,3), e₃ = (2,3), e₄ = (3,4). The incidence matrix is:


     e₁  e₂  e₃  e₄
1  [  1   1   0   0  ]
2  [  1   0   1   0  ]
3  [  0   1   1   1  ]
4  [  0   0   0   1  ]

Choosing the Right Representation

The choice of graph representation depends on the specific requirements of your problem and the operations you need to perform frequently.

Use Adjacency Matrix When:

  • The graph is dense (many edges).
  • You need to check quickly if there is an edge between any two vertices.
  • You need to perform matrix operations on the graph.
  • The graph is static (not changing frequently).

Use Adjacency List When:

  • The graph is sparse (few edges compared to the maximum possible).
  • You need to iterate over all edges efficiently.
  • You need to add or remove vertices frequently.
  • Memory efficiency is important.

Use Edge List When:

  • You need a simple representation.
  • The primary operation is to iterate over all edges.
  • Space efficiency is critical, and you don't need fast access to specific edges.
Example: Space Complexity Comparison

Consider a graph with 1000 vertices and 2000 edges:

  • Adjacency Matrix: Requires 1000² = 1,000,000 entries.
  • Adjacency List: Requires 1000 + 2 × 2000 = 5,000 entries (assuming each edge is stored in two adjacency lists for an undirected graph).
  • Edge List: Requires 2 × 2000 = 4,000 entries (assuming each edge is stored as a pair of vertices).

For this sparse graph, adjacency list and edge list are much more space-efficient than adjacency matrix.

Time Complexity Comparison

Operation Adjacency Matrix Adjacency List Edge List
Add Edge O(1) O(1) O(1)
Remove Edge O(1) O(degree) O(E)
Check if Edge Exists O(1) O(degree) O(E)
Find All Neighbors O(V) O(degree) O(E)
Add Vertex O(V²) O(1) O(1)
Remove Vertex O(V²) O(V + E) O(E)
Memory Usage O(V²) O(V + E) O(E)
