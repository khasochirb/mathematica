Graph Traversal

Understanding Depth-First Search (DFS) and Breadth-First Search (BFS)

Home » Learning Resources » Mathematics Roadmap » Graph Theory » Graph Traversal

Introduction to Graph Traversal

Graph traversal (also known as graph search) refers to the process of visiting (checking and/or updating) each vertex in a graph. Traversal algorithms are fundamental in graph theory and have numerous applications in computer science, network analysis, and optimization problems.

The two primary graph traversal techniques are:

  • Depth-First Search (DFS): Explores as far as possible along each branch before backtracking.
  • Breadth-First Search (BFS): Explores all vertices at the present depth before moving on to vertices at the next depth level.

Both DFS and BFS are complete algorithms, meaning they will find a solution if one exists, provided the graph is finite. However, they differ in their traversal pattern, implementation, and suitability for different problems.

Graph Representation for Traversal

Before discussing the traversal algorithms, it's important to understand how graphs are represented for traversal:

  • Adjacency List: For each vertex, store a list of its adjacent vertices. This is efficient for sparse graphs and is commonly used for traversal algorithms.
  • Adjacency Matrix: Store a matrix where element [i][j] indicates if there is an edge from vertex i to vertex j. While less space-efficient for sparse graphs, it provides O(1) access to check if an edge exists.
Example: Graph for Traversal

Consider the following undirected graph G:


    A --- B
    |     |
    |     |
    C --- D --- E
          |
          |
          F

Adjacency List Representation:

  • A: [B, C]
  • B: [A, D]
  • C: [A, D]
  • D: [B, C, E, F]
  • E: [D]
  • F: [D]

In subsequent examples, we'll use this graph to illustrate DFS and BFS traversals.

Depth-First Search (DFS)

Basic Concept

Depth-First Search (DFS) is an algorithm for traversing or searching tree or graph data structures. The algorithm starts at the root node (selecting some arbitrary node as the root in the case of a graph) and explores as far as possible along each branch before backtracking.

Algorithm

DFS can be implemented using either recursion or an explicit stack:

Recursive Implementation:

function DFS(G, v):

mark v as visited

process vertex v

for each neighbor w of v in G:

if w is not visited:

DFS(G, w)

Iterative Implementation (using a stack):

function DFS(G, start):

let S be a stack

S.push(start)

while S is not empty:

v = S.pop()

if v is not visited:

mark v as visited

process vertex v

for each neighbor w of v in G (in reverse order):

if w is not visited:

S.push(w)

Time and Space Complexity

  • Time Complexity: O(V + E), where V is the number of vertices and E is the number of edges in the graph. This is because, in the worst case, we need to visit all vertices and explore all edges.
  • Space Complexity: O(V), as in the worst case, the stack (or recursion stack) may need to hold all vertices.

Properties of DFS

  • DFS will explore one path in the graph to its full depth before backtracking to explore other paths.
  • DFS might not find the shortest path to a node.
  • DFS is often preferred when we want to visit all nodes in a graph, especially if the solution is far from the root.
Example: DFS Traversal

Let's perform a DFS traversal on the graph from the previous example, starting from vertex A:

Step 1: Start with A. Mark A as visited and process it. A's neighbors are B and C.

Step 2: Move to B (arbitrarily chosen). Mark B as visited and process it. B's neighbors are A (already visited) and D.

Step 3: Move to D. Mark D as visited and process it. D's neighbors are B (already visited), C, E, and F.

Step 4: Move to C. Mark C as visited and process it. C's neighbors are A (already visited) and D (already visited).

Step 5: Backtrack to D and move to E. Mark E as visited and process it. E's neighbor is D (already visited).

Step 6: Backtrack to D and move to F. Mark F as visited and process it. F's neighbor is D (already visited).

DFS traversal order: A, B, D, C, E, F

Applications of DFS

  • Topological Sorting: Used to order the vertices of a directed acyclic graph (DAG) such that for every directed edge u → v, vertex u comes before v in the ordering.
  • Finding Connected Components: DFS can be used to find all connected components in an undirected graph by running DFS from each unvisited vertex.
  • Detecting Cycles: A graph has a cycle if, during a DFS traversal, we encounter a vertex that is already in the recursion stack.
  • Solving Puzzles: DFS is often used to solve puzzles with only one solution, such as mazes or the 8-puzzle problem.
  • Finding Paths: DFS can be used to find a path between two vertices, though it might not be the shortest path.

Time Complexities for Common DFS Applications:

- Topological Sort: O(V + E)

- Finding Connected Components: O(V + E)

- Cycle Detection: O(V + E)

Breadth-First Search (BFS)

Basic Concept

Breadth-First Search (BFS) is an algorithm for traversing or searching tree or graph data structures. Unlike DFS, which explores as far as possible along a branch before backtracking, BFS explores all vertices at the present depth level before moving on to vertices at the next depth level.

Algorithm

BFS is typically implemented using a queue:

function BFS(G, start):

let Q be a queue

mark start as visited

Q.enqueue(start)

while Q is not empty:

v = Q.dequeue()

process vertex v

for each neighbor w of v in G:

if w is not visited:

mark w as visited

Q.enqueue(w)

Time and Space Complexity

  • Time Complexity: O(V + E), where V is the number of vertices and E is the number of edges in the graph. This is because, in the worst case, we need to visit all vertices and explore all edges.
  • Space Complexity: O(V), as in the worst case, the queue may need to hold all vertices.

Properties of BFS

  • BFS visits all vertices at the current level before moving to the next level.
  • BFS will find the shortest path to a node in terms of the number of edges, making it ideal for shortest path problems in unweighted graphs.
  • BFS is often preferred when the solution is likely to be close to the root or when we want to find the shortest path in an unweighted graph.
Example: BFS Traversal

Let's perform a BFS traversal on the graph from the earlier example, starting from vertex A:

Step 1: Start with A. Mark A as visited, process it, and enqueue it. Queue: [A]

Step 2: Dequeue A. A's neighbors are B and C. Mark them as visited, process them, and enqueue them. Queue: [B, C]

Step 3: Dequeue B. B's neighbors are A (already visited) and D. Mark D as visited, process it, and enqueue it. Queue: [C, D]

Step 4: Dequeue C. C's neighbors are A (already visited) and D (already visited). Queue: [D]

Step 5: Dequeue D. D's neighbors are B (already visited), C (already visited), E, and F. Mark E and F as visited, process them, and enqueue them. Queue: [E, F]

Step 6: Dequeue E. E's neighbor is D (already visited). Queue: [F]

Step 7: Dequeue F. F's neighbor is D (already visited). Queue is now empty, so we're done.

BFS traversal order: A, B, C, D, E, F

Applications of BFS

  • Shortest Path (Unweighted Graph): BFS can find the shortest path between two vertices in an unweighted graph.
  • Connected Components: Like DFS, BFS can be used to find connected components in an undirected graph.
  • Two-Coloring: BFS can be used to determine if a graph is bipartite by coloring the graph such that no two adjacent vertices have the same color.
  • Network Broadcasting: In network algorithms, BFS can be used to simulate broadcasting from a source to all nodes.
  • Web Crawlers: BFS is often used in web crawlers to systematically explore web pages level by level.

Time Complexities for Common BFS Applications:

- Shortest Path (Unweighted): O(V + E)

- Bipartite Graph Check: O(V + E)

- Finding Connected Components: O(V + E)

Comparison: DFS vs. BFS

Key Differences

Feature Depth-First Search (DFS) Breadth-First Search (BFS)
Exploration Pattern Explores as far as possible along a branch before backtracking Explores all vertices at the current level before moving to the next level
Data Structure Stack (or recursion stack) Queue
Space Complexity O(h) where h is the height of the tree or the maximum depth in the graph O(w) where w is the maximum width of the tree or graph
Path Finding Not guaranteed to find the shortest path Guaranteed to find the shortest path in unweighted graphs
Best Used For Topological sorting, cycle detection, maze solving, depth-first properties Shortest path problems, level-order traversal, network broadcasting

When to Use Which?

Use DFS when:

  • You need to find a path to a target and want to explore paths deeply before exploring alternatives.
  • You're working with problems like topological sorting or cycle detection.
  • Memory is a constraint and the graph is deep rather than wide.
  • You're solving puzzles with many possible dead ends (like a maze).

Use BFS when:

  • You need to find the shortest path in an unweighted graph.
  • You want to explore all neighbors at the current level before moving deeper.
  • You're working on problems like the shortest path or level-order traversal.
  • The graph is wide rather than deep.
Example: Shortest Path Comparison

Consider the task of finding a path from A to F in our example graph.

Using DFS (starting from A and prioritizing alphabetical order of neighbors):

  1. Start at A, visit B
  2. From B, visit D
  3. From D, visit F

Path: A → B → D → F (length 3)

Using BFS (starting from A):

  1. Start at A, enqueue neighbors B and C
  2. Dequeue B, enqueue D (neighbor of B)
  3. Dequeue C, no new neighbors to enqueue
  4. Dequeue D, enqueue E and F (neighbors of D)
  5. Path to F found: A → B → D → F (length 3)

In this case, both DFS and BFS found a path of length 3, which is the shortest path. However, this isn't always the case. If there were multiple paths to F, BFS would guarantee finding the shortest one, while DFS might find a longer path first.

Advanced Variations and Algorithms

Variations of DFS

  • Iterative Deepening DFS (IDDFS): Combines the space efficiency of DFS with the breadth-first search's completeness (finding the shallowest goal).
  • Bidirectional DFS: Runs two simultaneous DFS searches, one from the start node and one from the goal node, aiming to meet in the middle.
  • Depth-Limited Search: A variant of DFS where the search is limited to a specified depth.

Variations of BFS

  • Bidirectional BFS: Similar to bidirectional DFS but using BFS from both ends.
  • Uniform Cost Search (Dijkstra's Algorithm): A generalization of BFS for weighted graphs, where the queue is replaced with a priority queue.
  • A* Search: An extension of Dijkstra's algorithm that uses heuristics to guide the search towards the goal.

Applications of Advanced Traversal Algorithms

  • Dijkstra's Algorithm: Used for finding the shortest paths in a weighted graph with non-negative edge weights.
  • A* Search: Used in pathfinding and graph traversal to find the shortest path efficiently using heuristics.
  • Bellman-Ford Algorithm: Used for finding the shortest paths in a weighted graph that may contain negative edge weights.
  • Prim's and Kruskal's Algorithms: Used for finding the minimum spanning tree in a connected weighted graph.

Time Complexities for Advanced Algorithms:

- Dijkstra's Algorithm: O(V²) or O((V+E)logV) with a binary heap

- A* Search: O(E) in the worst case, but typically performs better with good heuristics

- Bellman-Ford Algorithm: O(V*E)
