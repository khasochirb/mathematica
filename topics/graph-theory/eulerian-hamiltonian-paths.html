Eulerian & Hamiltonian Paths

Exploring special paths and circuits in graphs

Home » Learning Resources » Mathematics Roadmap » Graph Theory » Eulerian & Hamiltonian Paths

Introduction

In graph theory, two classical problems involve finding special paths or circuits in graphs: the Eulerian path/circuit problem and the Hamiltonian path/circuit problem. Both problems have historical significance and numerous practical applications, but they differ fundamentally in their definitions and complexity.

Historical Background

The Eulerian path problem originated from the famous "Seven Bridges of Königsberg" problem, posed by Leonhard Euler in 1736. The city of Königsberg (now Kaliningrad, Russia) had seven bridges connecting four land masses. The question was whether one could walk through the city crossing each bridge exactly once. Euler proved this was impossible, laying the foundation for graph theory.

The Hamiltonian path problem is named after Sir William Rowan Hamilton, who invented a puzzle called the "Icosian game" in 1857. This puzzle involved finding a path along the edges of a dodecahedron that visits each vertex exactly once and returns to the starting point.

Despite their similar descriptions, these problems have very different computational complexities. The Eulerian path problem can be solved in linear time, while the Hamiltonian path problem is NP-complete, meaning no efficient algorithm is known for solving it in the general case.

Eulerian Paths and Circuits

Definitions

  • Eulerian Path: A path in a graph that visits every edge exactly once.
  • Eulerian Circuit: An Eulerian path that starts and ends at the same vertex.

Conditions for Existence

For an undirected graph:

  • Eulerian Circuit: A connected graph has an Eulerian circuit if and only if every vertex has an even degree.
  • Eulerian Path: A connected graph has an Eulerian path if and only if at most two vertices have odd degree. If exactly two vertices have odd degree, any Eulerian path must start at one of these vertices and end at the other.

For a directed graph:

  • Eulerian Circuit: A strongly connected graph has an Eulerian circuit if and only if every vertex has equal in-degree and out-degree.
  • Eulerian Path: A weakly connected graph has an Eulerian path if and only if:
    1. At most one vertex has (out-degree) - (in-degree) = 1 (this would be the starting vertex)
    2. At most one vertex has (in-degree) - (out-degree) = 1 (this would be the ending vertex)
    3. All other vertices have equal in-degree and out-degree

An undirected graph G has an Eulerian circuit if and only if:

1. G is connected (ignoring isolated vertices)

2. degree(v) is even for all vertices v in G

An undirected graph G has an Eulerian path if and only if:

1. G is connected (ignoring isolated vertices)

2. At most two vertices have odd degree

Finding Eulerian Paths and Circuits

There are efficient algorithms for finding Eulerian paths and circuits:

1. Hierholzer's Algorithm

  1. Start at any vertex (for a path, start at a vertex with odd degree if one exists).
  2. Follow edges and delete them as you go, until you get stuck (i.e., return to the starting vertex).
  3. If there are unused edges, find a vertex in your current circuit that has unused edges, start a new circuit from there, and splice it into the main circuit.
  4. Repeat until all edges are used.

2. Fleury's Algorithm

  1. Start at a vertex with odd degree (or any vertex if all have even degree).
  2. Follow edges, deleting them as you go, with the rule: only choose a bridge (an edge whose removal would disconnect the graph) if there is no alternative.
  3. Continue until all edges are used.
Example: Determining Eulerian Properties

Consider the following undirected graph:

Vertices: {A, B, C, D, E}

Edges: {(A,B), (A,C), (B,C), (B,D), (C,D), (C,E), (D,E)}

Let's check the degrees:

A: degree 2 (even)

B: degree 3 (odd)

C: degree 4 (even)

D: degree 3 (odd)

E: degree 2 (even)

Since there are exactly two vertices with odd degree (B and D), this graph has an Eulerian path but not an Eulerian circuit. Any Eulerian path must start at B and end at D (or vice versa).

A possible Eulerian path is: B-A-C-B-D-C-E-D

Applications of Eulerian Paths

  • DNA Fragment Assembly: In genomics, DNA fragments can be represented as vertices, with edges indicating overlap. An Eulerian path through this graph can help reconstruct the original DNA sequence.
  • Circuit Design: In electronic circuit design, Eulerian paths help design circuits that require each connection to be traversed exactly once.
  • Network Routing: Efficient routing protocols in computer networks can use Eulerian path concepts to optimize traffic flow.
  • Chinese Postman Problem: Finding the shortest path that traverses every edge at least once—closely related to Eulerian paths.

Hamiltonian Paths and Circuits

Definitions

  • Hamiltonian Path: A path in a graph that visits every vertex exactly once.
  • Hamiltonian Circuit (or Cycle): A Hamiltonian path that starts and ends at the same vertex, forming a cycle.

Conditions for Existence

Unlike Eulerian paths, there are no simple necessary and sufficient conditions for the existence of Hamiltonian paths or circuits in general graphs. However, there are several sufficient conditions:

1. Dirac's Theorem: If G is a simple graph with n vertices (n ≥ 3), and the degree of each vertex is at least n/2, then G has a Hamiltonian circuit.

2. Ore's Theorem: If G is a simple graph with n vertices (n ≥ 3), and for every pair of non-adjacent vertices u and v, the sum of their degrees is at least n, then G has a Hamiltonian circuit.

3. Complete Graphs: Every complete graph with n vertices (n ≥ 3) has a Hamiltonian circuit.

Dirac's Theorem: If G is a simple graph with n vertices (n ≥ 3), and:

degree(v) ≥ n/2 for all vertices v in G

then G has a Hamiltonian circuit.

Ore's Theorem: If G is a simple graph with n vertices (n ≥ 3), and:

degree(u) + degree(v) ≥ n for all non-adjacent vertices u, v in G

then G has a Hamiltonian circuit.

Finding Hamiltonian Paths and Circuits

Since the Hamiltonian path problem is NP-complete, there is no known polynomial-time algorithm for finding Hamiltonian paths in general graphs. However, several approaches can be used:

1. Brute Force: For small graphs, try all possible permutations of vertices to check if any form a valid Hamiltonian path or circuit.

2. Backtracking: Start from a vertex and explore paths, backtracking when reaching a dead end. Optimize with pruning techniques.

3. Dynamic Programming: For moderate-sized graphs, use the Held-Karp algorithm, which solves the problem in O(n² × 2ⁿ) time.

4. Heuristic Algorithms: For large graphs, use heuristic approaches such as genetic algorithms, simulated annealing, or ant colony optimization to find approximate solutions.

Example: Checking Hamiltonian Properties

Consider a cycle graph with 5 vertices {A, B, C, D, E} and edges {(A,B), (B,C), (C,D), (D,E), (E,A)}:

1. This graph already forms a Hamiltonian circuit: A-B-C-D-E-A

2. Any path that includes all vertices but doesn't return to the start is a Hamiltonian path, e.g., A-B-C-D-E

Now, if we add the chord (A,C), the graph still has Hamiltonian circuits, but more options exist, e.g., A-B-C-D-E-A or A-C-D-E-A.

If we remove edge (B,C), can we still have a Hamiltonian circuit? Yes, using the chord: A-B-A-C-D-E-A (which isn't valid because we visit A twice) or A-E-D-C-A-B-A (also invalid). So, after this removal, there is no Hamiltonian circuit, though Hamiltonian paths still exist, e.g., B-A-C-D-E.

Applications of Hamiltonian Paths

  • Traveling Salesman Problem (TSP): Finding the shortest Hamiltonian circuit in a weighted graph, crucial for logistics and route planning.
  • Knight's Tour: Finding a sequence of moves of a knight on a chessboard such that the knight visits every square exactly once.
  • Vehicle Routing: Planning routes for delivery vehicles to visit all customers efficiently.
  • Genome Sequencing: In bioinformatics, reconstructing the order of DNA fragments.
  • Circuit Design: Designing efficient electronic circuits where each component must be accessed once.

Comparison and Special Cases

Eulerian vs. Hamiltonian Paths

Feature Eulerian Paths/Circuits Hamiltonian Paths/Circuits
Visits what? Every edge exactly once Every vertex exactly once
Existence criteria Simple, based on vertex degrees Complex, no simple necessary and sufficient condition
Computational complexity Polynomial time (O(E), where E is the number of edges) NP-complete
Algorithm approach Direct, constructive algorithms Backtracking, dynamic programming, heuristics

Special Cases and Interesting Facts

  • Complete Graphs: Every complete graph with n ≥ 3 vertices has both Eulerian and Hamiltonian circuits if n is odd. If n is even, it has Hamiltonian circuits but no Eulerian circuit.
  • Cycle Graphs: Every cycle graph has both an Eulerian circuit and a Hamiltonian circuit (they are identical).
  • Bipartite Graphs: A bipartite graph can have an Eulerian circuit only if all vertices have even degree. It can have a Hamiltonian circuit only if the two partitions have the same number of vertices.
  • Path Graphs: A path graph has an Eulerian path (the path itself) but no Eulerian circuit. It also has a Hamiltonian path but no Hamiltonian circuit.
Example: Complete Graph K₅

Consider the complete graph K₅ with 5 vertices {A, B, C, D, E} where every vertex is connected to every other vertex.

Each vertex has degree 4 (even), so K₅ has an Eulerian circuit.

By definition, K₅ has Hamiltonian circuits, e.g., A-B-C-D-E-A.

K₅ has (5-1)!/2 = 12 distinct Hamiltonian circuits, considering that circuits can be traversed in either direction.

Practical Algorithms and Examples

Implementing Hierholzer's Algorithm for Eulerian Circuits

Here's a pseudocode implementation of Hierholzer's algorithm for finding an Eulerian circuit in an undirected graph:

Function FindEulerianCircuit(Graph G):

// Check if an Eulerian circuit exists

if G is not connected or any vertex has odd degree:

return "No Eulerian circuit exists"

// Create a copy of the graph's adjacency list

Let adjacency = copy of G's adjacency list

// Start with an empty stack and an empty circuit

Let stack = empty stack

Let circuit = empty list

// Start from any vertex

Let current = any vertex in G

stack.push(current)

while stack is not empty:

if adjacency[current] is empty:

circuit.prepend(current)

current = stack.pop()

else:

Let next = any vertex in adjacency[current]

stack.push(current)

// Remove the edge (current, next)

adjacency[current].remove(next)

adjacency[next].remove(current)

current = next

return circuit

Backtracking for Hamiltonian Paths

Here's a pseudocode implementation of a backtracking algorithm for finding a Hamiltonian path:

Function FindHamiltonianPath(Graph G):

Let n = number of vertices in G

Let path = array of size n, initialized with -1

path[0] = any vertex in G // Start from any vertex

if HamiltonianPathUtil(G, path, 1):

return path

else:

return "No Hamiltonian Path exists"

Function HamiltonianPathUtil(Graph G, path, position):

if position == n:

return true // All vertices are included

for each vertex v in G:

if CanBeAdded(v, G, path, position):

path[position] = v

if HamiltonianPathUtil(G, path, position+1):

return true

// Backtrack

path[position] = -1

return false

Function CanBeAdded(vertex v, Graph G, path, position):

// Check if the vertex is already in the path

if v exists in path:

return false

// Check if there's an edge from the previous vertex to this vertex

if position == 0 or G has an edge from path[position-1] to v:

return true

return false

Example: The Traveling Salesman Problem

The Traveling Salesman Problem (TSP) is a classic application of Hamiltonian circuits. Given a list of cities and the distances between each pair of cities, the task is to find the shortest possible route that visits each city exactly once and returns to the origin city.

Consider a small example with 4 cities and the following distance matrix:


    A   B   C   D
A   0  10  15  20
B  10   0  35  25
C  15  35   0  30
D  20  25  30   0

For this small instance, we can use a brute force approach to find the optimal solution by checking all possible Hamiltonian circuits:

  • A-B-C-D-A: 10 + 35 + 30 + 20 = 95
  • A-B-D-C-A: 10 + 25 + 30 + 15 = 80
  • A-C-B-D-A: 15 + 35 + 25 + 20 = 95
  • A-C-D-B-A: 15 + 30 + 25 + 10 = 80
  • A-D-B-C-A: 20 + 25 + 35 + 15 = 95
  • A-D-C-B-A: 20 + 30 + 35 + 10 = 95

The optimal solution is either A-B-D-C-A or A-C-D-B-A, both with a total distance of 80.
