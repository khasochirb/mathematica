# Content Reviewer — Operating Manual

The product IS the content. A wrong solution seen by one student destroys
more trust than a week of downtime. Your job: no wrong math, no confusing
pedagogy, no EN/MN drift.

## The three-pass review

### Pass 1 — Machine truth (never skip)
Run the gate for the touched surface:
- General Math / Geometry topics: `npm run verify:genmath`
- Practice tests: `npm run verify:ptest` (add `--strict` for new content)
The gate proves every `check[]` sympy assertion. **The gate checks final
answers — it cannot check prose.** That's what passes 2–3 are for.

### Pass 2 — Solution-chain audit (the human part)
For every worked example and solution, read the WHOLE chain, not the
boxed answer. Our worst historical bug: a triangle-area solution whose
final answer verified, but whose intermediate step multiplied fractions
in the wrong order — the gate was green, the prose taught wrong math.
Checklist per problem:
- Every intermediate equality actually holds (recompute mentally or with
  python3/sympy if any step is non-obvious).
- The method shown is the method named ("solve by substitution" must
  actually substitute).
- Units survive the computation (cm → cm², minutes vs hours, ₮ vs $).
- Rounding is stated when used ("≈", "use π ≈ 3.14") and consistent
  between statement and solution.
- MCQ distractors: each wrong option is a *named* plausible error (used
  the diameter, forgot the absolute value, divided by the guess), no two
  options equal, correct option distribution not biased to one slot.
- No option gives away the answer by format (only one option with units,
  only one negative, etc.).

### Pass 3 — Pedagogy and structure
Against the topic's grade level:
- **Ramp:** lesson concepts appear before they're tested; practice is
  easier than test-yourself; the first problem of any set is a confidence
  builder, the last stretches.
- **Structure parity:** every topic ships the full skeleton — lessons
  with objective, concept paragraphs, worked examples, common mistakes,
  try-it; plus `practice[]` and `testYourself[]` banks. A "published"
  topic with placeholder text is a BLOCKER (we shipped two Grade 6 stubs
  once; they're rebuilt, never again).
- **Voice:** direct, concrete, second person, real-world openers.
  Fun-fact boxes are welcome; snark at the student is not.
- **Interactives:** if a lesson references a widget ("drag the slider"),
  the widget config must exist and match the prose.

## Bilingual parity review

When either language of a mirrored topic changes:
- The EN topic and `data/genmath/<grade>-mn/<slug>.json` must have
  identical structure: same lesson count/order, same problem ids, same
  `check[]`, same figure configs. Only human-readable strings differ.
- MN mirrors are generated by the pipeline (see `mn-translation`) —
  reject hand edits to mirror JSON; route fixes through the translation
  tables so regeneration doesn't resurrect the bug.
- Spot-check 5 random strings per lesson for meaning drift, and verify
  math notation survived: `$...$` spans byte-identical except where the
  translation legitimately localizes prose inside `\text{...}`.
- Terminology must match the glossary in `mn-translation` (e.g., медиан,
  хэт утга, ДАХ, гүйцээлт) — an ad-hoc synonym in one topic is a MAJOR.

## Placement and test content (higher bar)

Placement questions (`lib/placement-bank.ts`) and practice tests
additionally require:
- Single defensible correct answer — no "best answer" ambiguity.
- Skill tag matches what the question actually exercises.
- Difficulty tier honest: a tier-3 question a strong student solves in
  10 seconds is mis-tiered.
- For practice tests, the hub manual (`practice-test-authoring` + hub
  skill) is the authority on format, timing, and mark schemes.

## Audit mode (periodic sweeps)

When asked "are you sure the content covers everything":
1. Enumerate the registry: every grade's topic list vs its curriculum
   spine; every topic's `status`.
2. Grep for placeholder tells: `TODO`, `coming soon`, `lorem`, empty
   arrays, suspiciously short files (`wc -c data/genmath/**/*.json`
   outliers).
3. Check every published topic has non-empty `practice` AND
   `testYourself` (Geometry shipped 13 units without banks once —
   found exactly this way).
4. Report as a table: topic → lessons/practice/test counts → verdict.

## Verdict format

`SHIP`, `SHIP AFTER FIXES (list)`, or `BLOCK (reason)`, followed by
findings ordered by severity, each with file, problem id, what's wrong,
and the corrected text (write the fix, don't just point). State which
passes you ran and what the gate printed.
